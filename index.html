Initializing Git

Install and create the environment for using Git on one's computer
  1. Download Git
  2. Click next through all prompts (except the path to enable Git to run in Windows CMD as well)
  3. Launch Git
  4. Add one's email address to Git
    • Type: "git config --global user.email 'email_address@example.com'"
  5. Add one's user name to Git
    • Type: "git config --global user.name 'One's username'"

Initializing Git repositories

Create repositories on one's local system and adding Git to the project folder
  1. Create the repository for the project
  2. Open that folder in Git Bash
    • Either type: "cd directory1/directory2/myApplication" or just right-click on the folder and click "Git Bash Here"
  3. Create one's .git file and initialize git in the folder
    • Type: "git init"
  4. Create the repository on Github
    • Copy the link of the remote repository as well, we will need that for the next step
  5. Add the remote repository location to your local Git repository
    • Type: "git remote add origin repository-url"

Adding, committing, and pushing

Majority of the development cycle, the main thing users will be doing with Git
  1. Add whichever files you want to save to Github into the staging area
    • Type: "git add filename.extension"
    • You can also exclude files by creating a .gitignore file and typing the filenames in there
    • You can also add all files (excluding those in the .gitignore file) at once by typing: "git add ."
  2. Commit your changes that you have added to the staging area
    • Type: "git commit -m 'Type a <50 character description here'"
      • The "-m" flag is so you can commit without going into the commit mode, which makes this step much faster
  3. Push your commit(s) to Github
    • Type: "git push -u origin master"
      • The "-u" flag is so when you pull, it makes it so you can do an argumentless git pull

Cloning, forking, and pulling

The first steps to contribute to an active collaborative project
  1. Generate an SSH key for the next few steps, and add it to the ssh-agent
  2. Fork the repository on Github's site
  3. Copy the repository on Github to your local system by cloning
    • Type: "git clone https://github.com/username/repository-name.git"
  4. To update from the master branch if it gets updated, then pull from it
    • Type: "git pull"
  5. Code, add, commit, and push
  • BONUS: Save your changes you are working on and work on another (perhaps more pressing) issue
    • To stash it away and save it, type: "git stash filename.extension
    • To continue working on it after working on the other issue, type: "git stash apply filename.extension

Branching, merging, and checking out

To work with other users, these are crucial skills to have in a team environment
  1. Type: "git branch" to list all the branches for the project on your local machine
  2. Create a new branch
    • Type: "git branch branch_name
  3. Switch to an existing branch
    • Type: "git checkout branch_name
  4. Merge the branch you have been working on with the master
    • Type: "git merge branch_name
  • BONUS: You can also create a branch and switch to it at the same time, type: "git checkout -b branch_name

Staying up to date, tracking changes, and reverting to a previous commit

Crucial practices to keep track of what is happening to the project
  1. Pull from the master or a particular branch
    • Type: "git pull"
    • For a particular branch, type: "git pull origin branch_name
  2. See what files have been changed and what is being tracked
    • Type: "git status"
    • If you want to see how much each file has changed by the number of lines, type: "git diff --stat"
  3. See the commit history
    • Type: "git log"
  4. Revert/Checkout to a previous commit version
    • git checkout 3427asdhj69123examplehash123912
