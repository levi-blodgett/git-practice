Install and create an environment for utilizing Git on your computer
  1. Download Git
  2. Click next through all prompts (except the path to enable Git to run in Windows CMD as well)
  3. Launch Git
  4. Add your email address to Git
    • git config --global user.email 'email_address@example.com'
  5. Add your user name to Git
    • git config --global user.name 'your username'
  • BONUS: You can also change the passphrase for your private key by typing:
    • ssh-keygen -f ~/.ssh/id_rsa -p
Create repositories on your local system and add Git to the project folder
  1. Create the repository for the project
  2. Open that folder in Git Bash
    • Either type: cd directory1/directory2/myApplication or just right-click on the folder and click "Git Bash Here"
  3. Create your .git file and initialize git in the folder
    • git init
  4. Create the repository on GitHub
    • Copy the link of the remote repository as well, we will need that for the next step
  5. Add the remote repository location to your local Git repository
    • git remote add origin repository-url
Begin your project, just these three commands alone are 90% of a developer's use of Git
  1. Add whichever files you want to save to GitHub into the staging area
    • git add filename.extension
    • You can also exclude files by creating a .gitignore file and typing the filenames in there
    • You can also add all files in current directory (excluding those in the .gitignore file) by typing: git add .
  2. Commit your changes that you have added to the staging area
    • git commit -m 'Type a <50 character description here'
      • The -a flag also automatically stages files that have been modified and deleted, but new files you have not told Git about are not affected.
  3. Push your commit(s) to GitHub
    • git push -u origin master
Start contributing to an active collaborative project after employing these techniques
  1. Generate an SSH key, add it to the ssh-agent, and add it to your GitHub account for the next few steps.
  2. Fork the repository by navigating to your desired repository on GitHub and clicking "Fork" in the top-right corner
  3. Copy your fork of the repository from GitHub onto your local system by cloning
    • git clone https://github.com/username/repository-name.git"
  4. To update from the master branch if it gets updated, then pull from it
    • git pull
  5. Code, add, commit, and push
  • BONUS: Save your changes you are working on and work on another (perhaps more pressing) issue
    • To stash it away and save it: git stash filename.extension
    • To continue working on it after working on the other issue: git stash apply filename.extension
Learn how to navigate enterprise-level workflow to actively contribute to a complicated project
  1. git branch to list all the branches for the project on your local machine
  2. Create a new branch
    • git branch branch_name
  3. Switch to an existing branch
    • git checkout branch_name
  4. Merge the branch you have been working on with the master
    • git merge branch_name
  • BONUS: You can also create a branch and switch to it simultaneously
    • git checkout -b branch_name
Stay up to date in a team environment, manage and track a project's workflow and structure
  1. Pull from the master or a particular branch
    • git pull
    • For a particular branch: git pull origin branch_name
  2. View what files have been changed and what is being tracked
    • git status
    • If you want to view how much each file has changed by the number of lines: git diff --stat
  3. View the commit history
    • git log
      • Type q to exit out of log mode.
    • If you want to format it to be easier to read, try the following flags:
      • git log --oneline
      • git log --pretty=format : "%h - %an, %ar : %s"
  4. Revert/Checkout to a previous commit version
    • git checkout 3427asdhj69123examplehash123912
